# PawsConnect — Claude Code Guide

## What This Is

PawsConnect is a pet adoption platform that connects adopters with shelters. Built with Laravel 12, Inertia.js, and React 19.

## Stack

| Layer | Technology |
|-------|-----------|
| Backend | PHP 8.2+, Laravel 12, Laravel Fortify, Laravel Sanctum |
| Frontend | React 19, TypeScript 5.7, Inertia.js 2 |
| Styling | Tailwind CSS 4, shadcn/ui (Radix UI) |
| Database | MySQL (production), SQLite (tests) |
| Auth | Laravel Fortify (web) + Laravel Sanctum (API tokens) |
| Testing | Pest 3.8 (70 % coverage minimum) |
| CI | GitHub Actions (`.github/workflows/tests.yml`) |

## Running the Project

```bash
# First-time setup
composer install
cp .env.example .env
php artisan key:generate
php artisan migrate --seed
npm install

# Dev server (PHP + Vite + queue worker — all at once)
composer dev

# Run tests
composer test

# PHP code style
composer lint          # auto-fix
composer test:lint     # dry-run (used in CI)

# Frontend
npm run dev            # Vite dev server only
npm run build          # production bundle
npm run format:check   # Prettier check (read-only)
npm run lint:check     # ESLint check (read-only)
npm run types          # TypeScript check
```

## Role System

Roles live in `App\Enums\Role` and are enforced via the `role` middleware alias → `EnsureRole`.

| Role | Value | Can Do |
|------|-------|--------|
| `adopter` | `adopter` | Browse pets, apply, favorite, manage profile |
| `shelter_staff` | `shelter_staff` | Post/edit/delete own pets, review applications |
| `admin` | `admin` | Everything — plus user management, trash, analytics |

```php
// Protecting routes
Route::middleware('role:admin')->group(...);
Route::middleware('role:shelter_staff,admin')->group(...);
```

## Key Conventions

### PHP

- Controllers **never** contain inline closures — all logic lives in controller methods.
- Route closures in `web.php` are limited to trivial one-liners (simple `Inertia::render` with no data, simple redirects).
- Use `optional()` / null-safe `?->` for nullable relationships. Never assume a relationship is loaded.
- Activity logging via Spatie `activity()` helper — always include `causedBy()` and `withProperties()`.
- Rate limiters are registered in `AppServiceProvider` and referenced in routes by name (`throttle:applications`).

### Frontend

- All pages are in `resources/js/Pages/`. Inertia resolves them by the string passed to `Inertia::render()`.
- Shared data (auth user, role, flash) comes from `HandleInertiaRequests::share()`.
- `resources/js/actions/` and `resources/js/routes/` are **auto-generated** by Wayfinder — never edit them by hand. They are regenerated on `npm run build` or `npm run dev`.
- The quiz/matchmaking feature (`Quiz.tsx`, `Matches.tsx`, `matchmakingService.ts`, `mockPets.ts`) runs entirely client-side against static mock data — this is intentional design.
- Use `@/` alias for imports from `resources/js/`.

### Testing

- Tests use Pest 3 with an in-memory SQLite database — no external dependencies.
- Test factories: only `UserFactory` exists. Create model fixtures directly in tests or via seeders.
- Minimum 70 % code coverage enforced in CI (`pest --coverage --min=70`).
- Run a single test file: `php artisan test tests/Feature/PetListingTest.php`

## API

### Session API (web middleware)

Used by the React frontend. Returns JSON.

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/pets` | Filter/search pets |
| GET | `/api/pets/stats` | Aggregate counts |
| GET | `/api/pets/{id}` | Single pet (auth) |
| PATCH | `/api/pets/{id}` | Update pet (staff/admin) |
| DELETE | `/api/pets/{id}` | Soft-delete pet (staff/admin) |
| GET | `/api/favorites` | Authenticated user's favorites |
| POST | `/api/favorites/toggle` | Toggle a pet in favorites |
| GET | `/api/shelter-visits/available-slots` | Available visit slots (public) |
| GET | `/api/shelter-visits/booked-dates` | Booked dates (public) |

### REST API v1 (Sanctum token auth)

Prefix: `/api/v1`. Authorization: `Bearer <token>` header.

| Method | Path | Auth |
|--------|------|------|
| GET | `/api/v1/pets` | Public |
| GET | `/api/v1/pets/{id}` | Public |
| GET | `/api/v1/shelters` | Public |
| GET | `/api/v1/shelters/{id}` | Public |
| GET | `/api/v1/user` | Token |
| GET | `/api/v1/applications` | Token |
| GET | `/api/v1/applications/{id}` | Token |
| POST | `/api/v1/applications` | Token |

## Storage

Pet images are stored on the `public` disk by default (XAMPP: `public/storage/`). Configure `FILESYSTEM_DISK=s3` in `.env` to switch to AWS S3.

```bash
php artisan storage:link   # create public/storage symlink (run once after setup)
```
